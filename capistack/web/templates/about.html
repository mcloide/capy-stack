About CapiStack

A lightweight, single-app CI/CD service you run yourself.

TL;DR

TL;DR

CapiStack is a self-hosted CI/CD platform that keeps things simple, secure, and transparent. It deploys a single app from a Git repo (GitHub, GitLab, or any Git server) to your servers with minimal config. Built with Python + Flask + SQLAlchemy + RQ, using a straightforward pipeline: checkout ‚Üí build ‚Üí deploy ‚Üí post-deploy.

In plain terms, it‚Äôs an over-glorified git pull with a web UI, auth, and logs. And yes, the mascot is a cute capybara my youngest picked‚Äîshe was too cute to say no. ü´∂

What it does
  • Single-app deploys from a browser (GitHub/GitLab or generic Git).
  • Select ref: branch, tag, or release.
  • Live progress via SSE; background jobs with RQ/Redis.
  • History: keeps the last 10 deployments with status, user, ref, timestamps.
  • Secrets: .env for config + encrypted secrets at rest (Fernet).
  • Auth modes (.env): none ¬∑ basic ¬∑ oauth (GitHub/GitLab/OIDC).
  • Database: Postgres or MySQL (SQLAlchemy + Alembic).
  • Runner: local by default; optional Docker isolation.
How it works (pipeline)
  1. Preflight ‚Äî validate repo, env, secrets.
  2. Checkout ‚Äî shallow clone and resolve commit SHA.
  3. Env inject ‚Äî merge project secrets into runtime env.
  4. Build (optional) ‚Äî run commands from capistack.yml (or skip).
  5. Deploy ‚Äî run deploy steps or ./deploy.sh.
  6. Post-deploy (optional) ‚Äî webhooks, cache clears, etc.
  7. Finalize ‚Äî persist logs/artifacts, enforce retention.
Why CapiStack
  • Focused: optimized for one service‚Äîfewer knobs.
  • Portable: Python, Flask, Redis, SQL; run anywhere.
  • Transparent: structured, streamable logs & explicit steps.
  • Safe by default: secrets encryption, minimal token scopes, clear auth.
Tech snapshot
  • Backend: Python 3.11+, Flask 3.x, SQLAlchemy 2.x, Alembic
  • Jobs/Logs: RQ + Redis, SSE/WebSocket for streaming
  • Git: provider APIs or git ls-remote fallback
  • UI: Jinja templates + minimal JS/HTMX
